nginx常用命令行

发表于2023-07-10|更新于2024-01-23|运维
|阅读量:

nginx命令行

  • 格式： nginx -s reload
  • 帮助： -? -h
  • 使用指定的配置文件： -c
  • 指定配置指令： -g
  • 指定运行目录： -p
  • 发送信号： -s

​ 立刻停止服务： stop 优雅的停止服务 quit 重载配置文件 reload 重新开始记录日志文件 reopen

  • 测试配置文件是否有语法错误： -t -T
  • 打印nginx的版本信息、编译信息等： -v -V

重载配置文件

修改配置文件中的值 vim ../conf/nginx.conf

修改后

退出vim编辑 :wq

重载配置文件后启动 ./nginx -s reload

没有报错就证明重启正常

热部署

查看nginx进程 ps -ef|grep nginx

如果我想更换最新版本的nginx，首先下载好最新的nginx，把现有nginx配置文件备份一下，因为我们更换的只是二进制文件，不会更换其他文件 cp ngnix ngnix.old

接下来把下载好的编译好的最新版本的nginx二进制文件复制到nginx安装目录中，替换掉正在的使用的nginx二进制文件

切换到最新版本的nginx文件目录 cd 文件目录

复制 cp -r nginx /usr/local/openresty/nginx/sbin/ -f

热部署kill -USR2 进程号

nginx master进程会用新的二进制文件生成新的worker进程，老的worker进程也在运行，他们会平滑的把所有请求过渡到新的进程中

向老进程发送信号，请优雅的关闭worker进程 kill -WINCH 进程号

老的master进程不会自动退出，为了允许我们做版本回退

日志切割

使用nginx -s reopen 或者 kill -USR1 进程号

比如access.log文件，我需要把以前的日志备份到另外的文件中，但是程序正常运行的

移动日志到另一个位置mv access.log bak.log

重新开始记录日志../sbin/nginx -s reopen

当然这种方法很不好用，实际使用中通常会一天或者一周进行日志切割

可以写成一个Bash脚本，先把要切割的文件备份一下，再执行-s reopen命令，把脚本放到crontab中定时执行
