nginx语法简介

基本语法

  1. 配置文件由指令和指令块构成

  2. 每条指令以;结尾，指令与参数间以空格符号分隔

  3. 指令块以{}大括号将多条指令组合在一起

  4. include语句允许组合多个配置文件以提升可维护性

  5. 使用#符号添加注释，提高可读性

  6. 使用$符号使用变量

  7. 部分指令的参数支持正则表达式

    http {
    include       mime.types;
    upstream thwp {
        server 127.0.0.1:8088;
    }
    server {
        listen                           443 hhtp2;
        #ngnix配置语法 limit_req_zone控制流速
        limit_req_zone $binary_remote_addr zone=one:10m rate=1r/s;
        location ~*\.(gif|jpg|jpeg)$ {
            proxy_cache my_cache;
            #3分钟后刷新
            expires 3m;proxy_cache_key $host$uri$is_args$args;
            proxy_cache_valid 200 304 302 1d;
            proxy_pass http://thwp;
        }

    时间单位：ms、s、m、h、d、w、M、y

    空间单位：bytes、k/K、m/M、g/G

配置的指令块

http

http大括号的里面的模块都是由http去解析去执行的。一个非http模块，如stream、mail是没办法解析里面的指令的

upstream

表示上游服务，ngnix需要与tomcat等等企业内网的其他服务交互的时候，可以定义upstream

server

对应的域名或者主域名

locaton

url表达式
