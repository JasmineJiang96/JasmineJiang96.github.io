【Lombok】介绍和常用注解

发表于2023-02-12|更新于2023-03-01|后端
|阅读量:

Lombok介绍

用于简化java Bean代码的java类库。

Lombok安装

  1. 在idea中安装Lombok插件
    File —> Settings —> Plugins —> Markplace —> 搜索lombok
    安装即可
  2. 在pom.xml文件添加依赖
    <dependency>
 <groupId>org.projectlombok</groupId>
 <artifactId>lombok</artifactId>
 <optional>true</optional>
</dependency>

Lombok常用注解

  • @Data
    自动生成Getter,Setter,equals,canEqual,hasCode,toString等方法,是@ToString @EqualsAndHashCode @Getter @Setter @RequiredArgsConstructor注解的集合。注解后在编译时会自动加进去。
  • @Value
    是@Data的不可变形式，相当于为属性添加final声明，只提供getter方法，而不提供setter方法。是@ToString @EqualsAndHashCode @Getter @RequiredArgsConstructor注解的集合。
  • @AllArgsConstructor
    使用后添加一个构造函数，该构造函数含有所有已声明字段属性参数
  • @NoArgsConstructor
    使用后创建一个无参构造函数
  • @Builder
    关于Builder较为复杂一些，Builder的作用之一是为了解决在某个类有很多构造函数的情况，也省去写很多构造函数的麻烦，在设计模式中的思想是：用一个内部类去实例化一个对象，避免一个类出现过多构造函数
  • @EqualsAndHashCode
    作用于类，生成 Equals 和 HashCode 方法
  • @ToString
    作用于类，可以生成一个字符串，按顺序打印类名称以及每个字段，并以逗号分隔，默认情况会打印所有的非静态字段，也可通过配置打印其他字段
  • @Getter
    作用于类，生成类所有的实例变量的 getter 方法；作用于变量，生成变量的 getter 方法
  • @Setter
    作用于类，生成类所有的实例变量的 setter 方法；作用于变量，生成该变量的 setter 方法
  • @NonNull
    作用于成员变量和参数中，添加空指针检查，标识不能为空，否则抛出NullPointerException(“param is marked @NonNull but is null”)
  • @Cleanup
    主要用来修饰 IO 流相关类, 会在 finally 代码块中对该资源进行 close()
  • @Synchronized
    给方法加上同步锁

外部注解

@CommonsLog，@Log，@JBossLog，@Log4j，@Log4j2，@Slf4j，@XSlf4j：门面模式实现日志注解，需要引用日志相关类库
@Slf4j
不使用注解之前，我们需要每个类都声明一个Logger，它从LoggerFactory中获取。

private final Logger logger = LoggerFactory.getLogger(currentClass.class);

类上标注@Slf4j，即可使用log.info()等方法。

@Slf4j
public class LogExample{
  public void test(){
    log.info("OK");
  }
}

缺点

  1. 代码可读性，可调试性低
  2. 必须安装第三方插件
  3. 影响升级，Lombok对于代码有很强的侵入性，就可能会影响我们对JDK的升级，如果我们要升级JDK，Lombok中不支持的话就会受到影响。
  4. 破坏封装性，滥用 getter 和 setter，违反了面向对象的封装特性。
