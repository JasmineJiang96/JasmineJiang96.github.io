【MongoDB】快速入门

发表于2022-11-21|更新于2023-03-02|数据库
|阅读量:

简介

MongoDB 是一个基于分布式文件存储的开源NoSQL数据库系统。由C++语言编写，旨在为WEB应用提供可扩展的高性能数据存储解决方案。MongoDB 提供了 面向文档 的存储方式，操作起来比较简单和容易，支持“无模式”的数据建模，可以存储比较复杂的数据类型，是一款非常流行的文档类型数据库 。

相关概念

MongoDB是非关系型数据库当中最像关系型数据库的，我们可以对比它与关系型数据库来进行学习。

  • 文档（Document） ：MongoDB 中最基本的单元，由 BSON 键值对（key-value）组成，类似于关系型数据库中的行（Row）。
  • 集合（Collection） ：一个集合可以包含多个文档，类似于关系型数据库中的表（Table）。
  • 数据库（Database） ：一个数据库中可以包含多个集合，可以在 MongoDB 中创建多个数据库，类似于关系型数据库中的数据库（Database）。
SQL MongoDB 解释/说明
database database 数据库
table collection 数据库表/集合
row document 数据记录行/文档
column field 数据字段/域
index index 索引
primary key Objectid 主键/对象ID

数据库操作

MongoDB 预留了几个特殊的数据库。

  • admin : 主要是保存 root 用户和角色。例如，system.users 表存储用户，system.roles 表存储角色。一般不建议用户直接操作这个数据库。将一个用户添加到这个数据库，且使它拥有 admin 库上的名为 dbAdminAnyDatabase 的角色权限，这个用户自动继承所有数据库的权限。一些特定的服务器端命令也只能从这个数据库运行，比如关闭服务器。
  • local : local 数据库是不会被复制到其他分片的，因此可以用来存储本地单台服务器的任意 collection。一般不建议用户直接使用 local 库存储任何数据，也不建议进行 CRUD 操作，因为数据无法被正常备份与恢复。
  • config : 当 MongoDB 使用分片设置时，config 数据库可用来保存分片的相关信息。
  • test : 默认创建的测试库，连接 mongod 服务时，如果不指定连接的具体数据库，默认就会连接到 test 数据库。

    创建数据库

    创建数据库，使用use命令去创建数据库，当插入第一条数据时会创建数据库，例如创建一个test数据库:
    > use jasmine
switched to db jasmine
> db.jasmine.insertOne({"name":"MongoDB教程"})
{
acknowledged: true,
insertedId: ObjectId("63f834f11451c87f264e3602")
}
    查看所有数据库，用show dbs命令
    > show dbs
admin    40.00 KiB
config  108.00 KiB
local    40.00 KiB
jasmine   40.00 KiB

    创建用户

    > db.createUser({user: "jasmine", pwd: "jasmine",roles: [{ role: "readWrite", db:"jasmine" }]})
{ ok: 1 }

    删除数据库

    切换到要删除的数据库，使用db.dropDatabase()命令，再用show dbs查看是否删除成功
    > use jasmine
switched to db jasmine
> db.dropDatabase()
{ ok: 1, dropped: 'jasmine' }
> show dbs
admin    40.00 KiB
config  108.00 KiB
local    40.00 KiB

    集合操作

    创建集合

    使用db.createCollection(name, options)方法来创建集合。
    参数说明
  • name: 要创建的集合名称
  • options: 可选参数, 指定有关内存大小及索引的选项
    options 可以是如下参数：
    字段 类型 描述
    capped 布尔 （可选）如果为 true，则创建固定集合。固定集合是指有着固定大小的集合，当达到最大值时，它会自动覆盖最早的文档。当该值为true时，必须指定size参数。
    size 数值 （可选）为固定集合指定一个最大值，即字节数。如果 capped 为 true，也需要指定该字段。
    max 数值 （可选）指定固定集合中包含文档的最大数量。

在插入文档时，MongoDB 首先检查固定集合的 size 字段，然后检查 max 字段。

> use test
switched to db test
> db.createCollection("jasmine")
{ "ok" : 1 }
> db.createCollection("mycol", { capped : true, size : 6142800, max : 10000 } )
{ "ok" : 1 }

如果要查看已有集合，可以使用show collections或show tables命令
在 MongoDB 中，你不需要创建集合。当你插入一些文档时，MongoDB 会自动创建集合。

删除集合

使用db.collection.drop()删除集合

 use test
switched to db test
> show collections
jasmine
> db.mycol.drop()
true
> show collections

文档操作

插入文档

3.2版本之前用 insert() 或 save() 方法向集合中插入文档，新版本中已废弃
db.collection.insertOne() 用于向集合插入一个新文档
db.collection.insertMany():向指定集合中插入多条文档数据

#  插入单条数据
> db.mycol.insertOne({"a": 1})
{
  acknowledged: true,
  insertedId: ObjectId("63fad6758b8820031471b084")
}
var document = ({"a": 1, "b": 1, "c": 1})
#  插入多条数据
> db.mycol.insertMany([{"b": 3}, {'c': 4}])
{
  acknowledged: true,
  insertedIds: {
    '0': ObjectId("63fad68d8b8820031471b085"),
    '1': ObjectId("63fad68d8b8820031471b086")
  }
}
# 定义成变量，再执行插入
> var document = ({"a": 1, "b": 1, "c": 1})
> db.mycol.insertOne(document)
{
  acknowledged: true,
  insertedId: ObjectId("63fad62c8b8820031471b083")
}

更新文档

updateOne 更新集合中 filter 匹配的单个文档。如果匹配到多个文档 updateOne 仅更新第一个匹配到的文档。

> db.mycol.updateOne({"a": 1}, {$set: {"b": 2, "c": 3}})
{
  acknowledged: true,
  insertedId: null,
  matchedCount: 1,
  modifiedCount: 1,
  upsertedCount: 0
}

updateMany 更新集合中所有匹配到的文档。

> db.mycol.updateMany({"a": 1}, {$set: {"b": 22, "c": 33}})
{
  acknowledged: true,
  insertedId: null,
  matchedCount: 2,
  modifiedCount: 1,
  upsertedCount: 0
}

3.2版本之前update()方法，replaceOne()方法

# 修改一条
db.mycol.update({"a": 1}, {$set: {"b": 2, "c": 3}})
# 修改多条
db.mycol.update({"a": 1}, {$set: {"b": 2, "c": 3}}, {multi:true})

> db.mycol.replaceOne({"a": 1}, {"a": 1, "b": 2, "c": 3})
{
  acknowledged: true,
  insertedId: null,
  matchedCount: 1,
  modifiedCount: 1,
  upsertedCount: 0
}

删除文档

# 删除一条
> db.mycol.deleteOne({"b": 3})
{ acknowledged: true, deletedCount: 1 }
# 删除多条
> db.mycol.deleteMany({"a": 1})
{ acknowledged: true, deletedCount: 2 }
# 删除一条
> db.mycol.findOneAndDelete({"c": 1})
{ _id: ObjectId("63fb6f6d5a53000099000714"), c: 1 }

查询文档

> db.mycol.find({"c": 1})
[
  { _id: ObjectId("63fb73095a53000099000716"), c: 1 },
  { _id: ObjectId("63fb730d5a53000099000717"), c: 1 }
]

db.collection.find().pretty()以格式化的方式来显示所有文档；
findOne()方法，只返回一个文档

条件操作符

操作 格式 SQL中的类似语句
等于 {:} 数where title = ‘value’
小于 {:{$lt:}} where likes < 50
小于或等于 {:{$lte:}} where likes <= 50
大于 {:{$gt:}} where likes > 50
大于或等于 {:{$gte:}} where likes >= 50
不等于 {:{$ne:}} where likes != 50
  • 条件查询，查询 c 等于 1 的数据
    db.mycol.find({"c": 1})
  • 条件查询，查询 c 大于 1 的数据
    db.mycol.find({"c": {$gt: 1}})
  • AND条件可以通过在find()方法传入多个键，以逗号隔开来实现，例如查询a等于1并且c大于2的所有文档；
    db.mycol.find({"a":1, "c": {$gt: 2}})
  • OR条件可以通过使用$or操作符实现，例如查询a等于1或c等于3所有文档；
    db.mycol.find({$or:[{"a":1, "c": 3}]})
  • AND 和 OR条件的联合使用,例如查询b等于2 并且a等于1或c等于3所有文档；
    db.mycol.find({"b": 2, $or:[{"a":1, "c": 3}]})

    其他操作

    Limit和Skip操作

  • 读取指定数量的文档，可以用limit()方法
    db.collection.find().limit(NUMBER)
  • 跳过指定数量的文档来读取，可以使用skip()方法
    db.collection.find().limit(NUMBER).skip(NUMBER)
  • 从第二条开始，查询mycol集合中的2条数据；
    db.mycol.find().limit(2).skip(1);

    排序

    使用sort()方法来排序，参数指定排序的字段，使用1和-1来指定排序方式，1为升序，-1为降序；
    db.collection.find().sort({KEY:1})
    按mycol集合中文档的a字段降序排列；
    db.mycol.find().sort({a: -1})

    索引

    使用createIndex()方法来创建索引
    db.collection.createIndex(keys, options)
    options包含下列参数：
  • background：建索引过程会阻塞其它数据库操作，设置为true表示后台创建，默认为false
  • unique：设置为true表示创建唯一索引
  • name：指定索引名称，如果没有指定会自动生成
    给a和b字段创建索引，1表示升序索引，-1表示降序索引，指定以后台方式创建；
    > db.mycol.createIndex({"a": 1, "b": -1}, {background: true})
a_1_b_-1
    查看mycol集合中已经创建的索引
    > db.mycol.getIndexes()
[
{ v: 2, key: { _id: 1 }, name: '_id_' },
{ v: 2, key: { a: 1, b: -1 }, name: 'a_1_b_-1', background: true }
]

    聚合

  • 使用aggregate()方法，类似于SQL中的group by语句
    db.collection.aggregate(AGGREGATE_OPERATION)
  • 聚合中常用操作符：
操作符 描述
$sum 计算总和
$avg 计算平均值
$min 计算最小值
$max 计算最大值
  • 根据a字段聚合文档并计算文档数量，类似与SQL中的count()函数；
    > db.mycol.aggregate([{$group: {_id: "$a", sumCount: {$sum: 2}}}])
[
{ _id: 1, sumCount: 2 },
{ _id: null, sumCount: 4 },
{ _id: 2, sumCount: 3 }
]
  • 根据a字段聚合文档并计算c字段的平局值，类似与SQL中的avg()语句；
    > db.mycol.aggregate([{$group: {_id: "$a", avg_c: {$avg: "$c"}}}])
[
{ _id: null, avg_c: 2.3333333333333335 },
{ _id: 2, avg_c: 1 },
{ _id: 1, avg_c: 3.5 }
]
